# Google-Play-Store-App-Analysis-Feature-Engineering
Google Play Store App Analysis project applies EDA and Feature Engineering to clean data, explore trends in categories, ratings, installs, and reviews, and build new features for deeper insights. Demonstrates practical skills in Python, pandas, seaborn, and scikit‑learn.

📌 Project Overview
This project explores the Google Play Store dataset using Exploratory Data Analysis (EDA) and Feature Engineering (FE). The goal is to uncover insights about app categories, ratings, installs, and reviews while preparing the dataset for machine learning workflows.

🛠️ Key Steps
Data Cleaning: Handle missing values, duplicates, and inconsistent formats.

EDA: Visualize distributions, trends, and correlations across app features.

Feature Engineering: Create new features (e.g., popularity indicators, rating buckets, text‑based features).

Insights: Compare free vs. paid apps, analyze top categories, and study relationships between installs, ratings, and reviews.

📊 Technologies Used
Python

pandas, NumPy

seaborn, matplotlib

scikit‑learn (for preprocessing and feature engineering)

📈 Results
Identified top app categories by count and proportion.

Visualized numerical feature distributions with KDE plots.

Built recruiter‑ready charts (bar plots, pie charts) with annotations.

Engineered features to improve downstream modeling.
